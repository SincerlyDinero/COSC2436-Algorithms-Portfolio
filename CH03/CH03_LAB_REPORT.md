## Algorithm Summary

- How it works:  
Recursion solves a problem by having a function call itself with smaller inputs until it reaches a base case. The base case stops the recursive calls and allows the answers to return through the call stack.

- Time complexity: 
Depends on the recursive problem being solved.

- When to use it: 
Recursion is useful for divide-and-conquer algorithms, tree traversal, factorial calculations, and repeated subproblems.

## Reflection Questions

1. What is a base case in recursion?  
A base case is the condition that stops recursive calls from continuing forever.

2. What are advantages of recursion? 
Recursion can simplify code and make complex problems easier to solve.

3. What are disadvantages of recursion?  
Recursive functions can consume more memory and may cause stack overflow errors.
